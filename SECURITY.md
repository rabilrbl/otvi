# Security Policy

## Reporting a Vulnerability

Do not report security vulnerabilities in public GitHub issues.

Instead:

1. Use GitHub's private vulnerability reporting flow if it is enabled for the repository.
2. If private reporting is not available, contact the maintainer privately through GitHub before disclosing details publicly.

Include:

- affected version or commit
- impact summary
- reproduction steps or proof of concept
- any suggested mitigation

## Disclosure Expectations

- We will acknowledge receipt as soon as practical.
- We may ask follow-up questions to confirm scope and impact.
- Please allow time for investigation and remediation before public disclosure.

## Supported Release Guidance

Security fixes are evaluated against the latest supported release train and any actively maintained prerelease work on `main`.
